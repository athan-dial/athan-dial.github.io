Write `.claude/commands/split-source.md` — a Claude Code command that takes a source note path as input and writes atom notes.

The command accepts one argument: the path to the source note. Example invocation: `/split-source ~/Library/Mobile Documents/.../sources/slack/Slack - 2026-03-01 - Adoption-curve-framing.md`

The command prompt must instruct Claude to:

1. Read the source note at the given path. Extract:
   - Full title from frontmatter
   - The `type` field to determine `source_type` (e.g., `source/slack` → `source_type: slack`)
   - The full body text

2. Identify distinct concepts in the source note body. A distinct concept is:
   - A single claim, insight, or observation that can stand alone
   - Not a combination of two separate ideas
   - Expressible as a declarative sentence in 10-15 words

3. For each concept (assigned a sequential split_index starting at 1):
   a. Write a title as a declarative sentence (the concept)
   b. Write 2-3 sentences for the body:
      - Sentence 1: The concept restated with any necessary context
      - Sentences 2-3: Why this matters in the context of the original source material

   CRITICAL — ATOM-04 constraint, embed this exact instruction in the prompt:
   "For each atom: write a title as a declarative sentence. Then write 2–3 sentences: first restate the concept with context, then explain why it matters given what you know about the original source. Do NOT write atoms as telegraphic bullet points. Do NOT strip the 'why it matters' context."

4. Create atom note filename: sanitize the title to lowercase-hyphenated-slug, truncate at 60 chars, append `.md`. Example: `adoption-curves-plateau-at-40-percent.md`

5. Write each atom note directly to `~/Library/Mobile Documents/iCloud~md~obsidian/Documents/2B-new/700 Model Citizen/atoms/{filename}` with exact SCHEMA.md frontmatter (see interfaces above).

6. After writing all atoms, update the source note: change `content_status: raw` to `content_status: processed` in the source note's frontmatter. Use sed or read-modify-write pattern.

7. Report: N atoms created from the source note, atom titles listed, source note marked as processed.

CRITICAL constraints to embed in the command prompt:
- Write atoms directly to `atoms/` folder — NOT to vault root
- Every atom MUST have `tags: [type/atom]` — this is the ANM routing tag
- `atom_of` field MUST be a wikilink to the parent source note title: `[[Title of source note]]`
- `publishability` is always `private` for all atoms — never auto-publish
- Do NOT copy provenance fields from source note to atom — backlink via `atom_of` is sufficient
- Do NOT combine two concepts into one atom — if in doubt, split further
- `split_index` must be sequential starting from 1 — it is used for reconstruction ordering
