```bash
# Extract keywords from atom title (5+ char words)
ATOM_TITLE=$(grep '^title:' "$ATOM_PATH" | sed 's/^title: //' | tr -d '"')
KEYWORDS=$(echo "$ATOM_TITLE" | tr ' ' '\n' | grep -E '.{5,}' | head -5)

# Find themes with title overlap
for kw in $KEYWORDS; do
  grep -li "$kw" "$VAULT_THEMES"/*.md 2>/dev/null
done | sort -u | head -3

# Find atoms with overlapping tags
ATOM_TAGS=$(grep -A5 '^tags:' "$ATOM_PATH" | grep -oP '[\w/]+' | tail -n +2)
for tag in $ATOM_TAGS; do
  grep -l "$tag" "$VAULT_ATOMS"/*.md 2>/dev/null
done | sort -u | grep -v "$ATOM_PATH" | head -5
```

Write `.claude/commands/match-themes.md` — a Claude Code command that takes an atom note path as input and connects it to related vault themes.

The command accepts one argument: the path to the atom note. Example invocation: `/match-themes ~/Library/Mobile Documents/.../atoms/adoption-curves-plateau-at-40-percent.md`

The command prompt must instruct Claude to:

1. Read the target atom note. Extract:
   - Title
   - Tags array
   - Body content (to understand the concept)

2. Run grep-based candidate discovery across three passes:
   a. **Title keyword grep against themes/**: Extract words of 5+ characters from the atom title. For each keyword, grep `themes/*.md` case-insensitively for matches. Collect unique matching theme files.
   b. **Tag overlap grep against atoms/**: Extract tags from the atom's frontmatter. For each tag, grep `atoms/*.md` for files containing that tag. Exclude the current atom. Collect unique matching atom files.
   c. **Title keyword grep against atoms/**: Use the same keywords to grep `atoms/*.md` for atoms with title matches.

3. Evaluate candidates:
   - Rank by number of keyword overlaps (more overlaps = higher relevance)
   - Select at most 3 candidates total (across themes and atoms)
   - HARD CAP: Stop at 3 connections even if more candidates exist (THEME-03)
   - Prefer theme file candidates over atom candidates — themes are the aggregation layer

4. For each selected candidate (up to 3):
   - Write a one-sentence justification: why does this atom connect to that theme/atom?
   - The justification must be specific to the content — not generic ("this is related")

5. Add a `## Theme connections` section to the atom body (append, do not overwrite existing content):
   ```markdown
   ## Theme connections

   - [[Theme Title]] — [one-sentence justification]
   - [[Another Theme]] — [one-sentence justification]
   ```

6. For each connection that points to an existing theme note:
   - Read the theme note
   - Find or create the `## Related atoms` section
   - Add `- [[Atom Title]]` to the section if not already present
   - Write the updated theme note

7. For each connection where NO matching theme note exists in themes/ (e.g., the keyword match was just a conceptual inference):
   - Create a new theme note at `themes/{sanitized-theme-title}.md`
   - Use SCHEMA.md theme frontmatter (see interfaces above)
   - Body: `## Related atoms` section with the current atom wikilinked
   - Write directly to `themes/` folder

8. Report: Atom title, N connections found, theme note names updated/created, any notes that had no matches (0 connections is a valid result).

CRITICAL constraints to embed in the command prompt:
- ABSOLUTE MAX 3 connections per atom — enforce this hard cap in the prompt: "Add at most 3 wikilinks. If you find 4+ candidates, pick the 3 most relevant by keyword overlap count. Stop after 3."
- Each wikilink MUST have a one-sentence justification — no bare wikilinks
- The justification must be specific content reasoning, not "this atom is related to this theme"
- Do NOT create theme notes for concepts that are too narrow or unique to one atom (themes should aggregate 2+ atoms eventually)
- Theme notes written to `themes/` must have `tags: [type/theme]` for ANM routing
- If the atom already has a `## Theme connections` section, append to it — do not overwrite
