**`.claude/commands/scan-slack-intelligence.md`** — Claude Code command prompt:

Write a complete agentic task prompt that instructs Claude to:

1. Read `~/.model-citizen/slack-intelligence-state.json`. If absent, initialize it: `{"last_scan_ts": 0, "seen_ids": []}`.
2. Call the verified Slack MCP tool to fetch starred messages. Pass the verified parameter names from slack-mcp-params.md. Filter to items newer than `last_scan_ts` (if the API supports timestamp filtering) OR fetch the full list and filter in-code by `ts > last_scan_ts`.
3. For each starred message:
   a. Check if message `ts` (or equivalent ID field) is in `seen_ids` — if yes, skip.
   b. Extract: full message text, sender username, channel name, timestamp, thread_ts (if threaded), permalink, starred_reason (if available from API; if not available use empty string).
   c. Generate a title from the first 60 chars of message text, sanitized.
   d. Write vault source note to `~/Library/Mobile Documents/iCloud~md~obsidian/Documents/2B-new/700 Model Citizen/sources/slack/Slack - YYYY-MM-DD - {sanitized-title}.md` with exact SCHEMA.md frontmatter (see context interfaces above).
   e. Append the message `ts` to a local `new_ids` list.
4. After processing all messages: update state file atomically — read current state, merge `new_ids` into `seen_ids`, set `last_scan_ts` to current Unix timestamp, write new JSON object.
5. Report: N new notes created, M skipped (already seen), any errors.

CRITICAL constraints to embed in the command prompt:
- Notes go directly to `sources/slack/` — NOT to vault root (skip ANM dependency)
- Every note MUST have both `type: source/slack` (YAML field) AND `tags: [type/source/slack]` (Obsidian tag array)
- State file update is atomic: read full current JSON, build new JSON object, write the complete new file — do NOT append
- publishability is always `private` for source notes — no exceptions
- Filenames must sanitize the title: strip special chars, replace spaces with hyphens, truncate at 60 chars
