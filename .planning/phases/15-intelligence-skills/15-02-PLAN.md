   e. Append the message `id` to a local `new_ids` list.
7. After processing all messages: update state file atomically — build a new complete JSON object with merged `seen_ids` and current Unix timestamp as `last_scan_ts`, write the full file.
8. Report: N new notes created, M skipped (already seen), any errors.
