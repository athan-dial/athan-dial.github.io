Create `model-citizen/vault/SCHEMA.md` — the authoritative schema contract for the content intelligence pipeline. Document all 4 note types with complete frontmatter examples:

**Source notes** (`type: source/slack` or `type: source/outlook`):
- Required fields: title, type, created, content_status (default: raw), publishability (default: private), provenance (nested YAML object), tags
- Slack provenance (nested): channel, sender, timestamp, thread_ts, permalink, starred_reason
- Outlook provenance (nested): sender, subject, date
- Document that ALL source notes are always private — no exceptions

**Atom notes** (`type: atom`):
- Required fields: title, type, created, content_status (default: raw), publishability (default: private), atom_of (wikilink to parent source), split_index (integer, reconstruction order), source_type (slack/outlook/goodlinks/youtube), tags
- Document that atoms do NOT copy full provenance — they reference parent via atom_of backlink
- Document that source_type is a lightweight origin field enabling Dataview queries without traversing backlinks (satisfies ATOM-02 intent)

**Theme notes** (`type: theme`):
- Required fields: title, type, created, publishability (default: private), tags
- No content_status, no provenance — themes are structural aggregation hubs
- Body contains "## Related atoms" section with wikilinks

**Draft notes** (`type: draft`):
- Required fields: title, type, created, modified, content_status (default: draft), publishability (default: private), source_atoms (list of wikilinks), tags
- Document that publishability must be manually flipped to public — human approval gate (SYNTH-04)

**content_status values** (document the full lifecycle):
- raw → processed → draft → ready → archived

**publishability rules**:
- Default: private for ALL note types
- Promotion: manual edit of publishability field to public
- ALL source and atom notes are always private (Slack/Outlook content never public)

**Folder routing** section:
- sources/slack/ — source/slack notes
- sources/outlook/ — source/outlook notes
- atoms/ — atom notes
- themes/ — theme notes
- drafts/ — draft notes
- Document that routing uses ANM pattern matching on frontmatter type field
- Note: existing folders (concepts/, writing/, explorations/, ideas/) remain unchanged

**Concepts vs Themes distinction**: concepts/ = manually created frameworks; themes/ = programmatically created aggregation hubs from the intelligence pipeline.
