  <action>
1. Backup current layouts/ directory to layouts.blowfish.bak/ (custom Blowfish partials for reference)

2. Update config/_default/module.toml to replace Blowfish with Hugo Resume:
   ```toml
   [[imports]]
     path = "github.com/eddiewebb/hugo-resume"
   ```
   (Remove the Blowfish import entirely)

3. Run Hugo module cleanup commands in sequence:
   - `hugo mod clean` — Clear module cache
   - `hugo mod tidy` — Remove Blowfish from go.mod/go.sum
   - `hugo mod get` — Download Hugo Resume theme

4. Verify with `hugo mod graph` — Should show ONLY hugo-resume, NO blowfish

AVOID: Do not delete go.mod/go.sum manually. Let hugo mod tidy handle the cleanup to ensure checksums are correct.

WHY: Research identified "Incomplete Blowfish Removal" as a common pitfall. Running hugo mod tidy after updating module.toml ensures go.mod/go.sum are properly synchronized.
  </action>
  <verify>
    Run: `hugo mod graph | grep -i blowfish` returns empty (no matches)
    Run: `hugo mod graph | grep -i resume` returns hugo-resume entry
    Run: `ls layouts.blowfish.bak/` confirms backup exists
  </verify>
  <done>
    - module.toml contains only hugo-resume import (no Blowfish)
    - go.mod contains hugo-resume dependency (no Blowfish)
    - go.sum has valid checksums for hugo-resume
    - layouts.blowfish.bak/ backup directory exists
  </done>
